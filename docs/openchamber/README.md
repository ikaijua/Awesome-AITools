# OpenChamber Introduction

## What is OpenChamber?

OpenChamber is a rich desktop, browser, and mobile interface for the [OpenCode](https://opencode.ai) AI coding agent. It lets you review diffs, manage agents, run dev servers, and keep the big picture while your AI codes — the same session available across the terminal, tablet, and phone.

## Why use OpenChamber?

- **Cross-device continuity** — start in the TUI, continue on tablet/phone, return to the terminal, all in the same session.
- **Remote access** — use OpenCode from anywhere via the browser.
- **Familiarity** — a visual alternative for developers who prefer GUI workflows.

## Key Features

### Core (all versions)
- Branchable chat timeline with `/undo`, `/redo`, and one-click forks from earlier turns.
- Smart tool UIs for diffs, file operations, permissions, and long-running task progress.
- Voice mode with speech input and read-aloud responses.
- Multi-agent runs from one prompt with isolated worktrees for safe side-by-side comparisons.
- In-app Git workflows: identities, commits, PR creation, checks, and merges.
- GitHub-native workflows: start sessions from issues and pull requests with context attached.
- Plan/Build mode with a dedicated plan view.
- Inline comment drafts on diffs, files, and plans that can be sent back to the agent.
- Context visibility tools (token/cost breakdowns, raw message inspection, activity summaries).
- Integrated terminal and a built-in skills catalog.

### Web / PWA
- Tunnel access via Cloudflare `quick`, `managed-remote`, and `managed-local` modes.
- One-scan onboarding with tunnel QR and password URL helpers.
- Mobile-first chat controls, keyboard-safe layouts, and background notifications.
- Built-in self-update and restart flow.

### Desktop (macOS + Windows + Linux)
- Floating Mini Chat that stays always-on-top beside your editor or terminal.
- Multiple native windows for separate projects or sessions.
- Native notifications and one-click open in VS Code, Cursor, Terminal, Finder, and Explorer.
- Host switcher for local and remote instances, plus SSH remote access with port forwarding.

## Quick Start

1. Install and run [OpenCode](https://opencode.ai).
2. Download OpenChamber from the [releases page](https://github.com/openchamber/openchamber/releases) or visit [openchamber.dev](https://openchamber.dev).
3. Connect OpenChamber to your OpenCode instance (local or remote) and start working.
